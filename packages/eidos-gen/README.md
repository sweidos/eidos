# eidos-gen

[![npm](https://img.shields.io/npm/v/eidos-gen)](https://www.npmjs.com/package/eidos-gen)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](../../LICENSE)

> Generate typed `resource()` and `action()` declarations from an OpenAPI 3.x spec.

`eidos-gen` reads your OpenAPI JSON or YAML file and writes a TypeScript file with fully-typed [`@sweidos/eidos`](https://www.npmjs.com/package/@sweidos/eidos) resource and action declarations — ready to import and use without any hand-writing.

---

## Quick start

```bash
npx eidos-gen openapi.json
```

Writes `eidos.generated.ts` to your current directory. Import and use:

```ts
import { getProducts, createOrder, deleteProduct } from './eidos.generated'

// Already typed — just call them
const products = await getProducts.fetch()
await createOrder({ item: 'widget', qty: 2 })
await deleteProduct({ id: '123' })
```

## Install

```bash
# Run without installing
npx eidos-gen openapi.json

# Or install globally
npm install -g eidos-gen

# Or as a dev dependency
npm install -D eidos-gen
```

## Usage

```
eidos-gen <spec> [options]

Arguments:
  <spec>              Path to OpenAPI 3.x spec (JSON or YAML)

Options:
  --out, -o <file>    Output file path (default: eidos.generated.ts)
  --no-offline        Set offline:false on resource() declarations
  --eidos <pkg>       Import path for @sweidos/eidos (default: @sweidos/eidos)
  --help, -h          Show help
```

**Examples:**

```bash
# Basic — reads openapi.json, writes eidos.generated.ts
npx eidos-gen openapi.json

# Custom output path
npx eidos-gen openapi.yaml --out src/lib/api.ts

# Inside a monorepo, import from a local workspace package
npx eidos-gen openapi.json --eidos @mycompany/eidos

# Disable offline caching (resources will still work but won't be served from cache when offline)
npx eidos-gen openapi.json --no-offline
```

## What gets generated

Given an OpenAPI spec with paths:

```yaml
paths:
  /api/products:
    get:
      operationId: getProducts
      summary: List all products
      responses:
        '200':
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/Product'
    post:
      operationId: createProduct
      requestBody:
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateProductInput'
      responses:
        '201':
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Product'
  /api/products/{id}:
    delete:
      operationId: deleteProduct
      responses:
        '204': {}
```

`eidos-gen` writes:

```ts
// Generated by eidos-gen — edit function bodies freely, re-run to refresh declarations.
import { resource, action } from '@sweidos/eidos'

// ── Generated types ───────────────────────────────────────────────────────────

export interface Product {
  id: string
  name: string
  price?: number
}

export interface CreateProductInput {
  name: string
  price?: number
}

// ── Resources (GET) ───────────────────────────────────────────────────────────

/** List all products */
export const getProducts = resource('/api/products', { offline: true })

// ── Actions (POST / PUT / PATCH / DELETE) ─────────────────────────────────────

export const createProduct = action(
  async (payload: CreateProductInput): Promise<Product> => {
    const res = await fetch('/api/products', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
    })
    if (!res.ok) throw Object.assign(res, { status: res.status })
    return res.json() as Promise<Product>
  },
  { reliability: 'neverLose', name: 'createProduct' },
)

export const deleteProduct = action(
  async (payload: { id: string }): Promise<void> => {
    const res = await fetch(`/api/products/${payload.id}`, {
      method: 'DELETE',
    })
    if (!res.ok) throw Object.assign(res, { status: res.status })
  },
  { reliability: 'neverLose', name: 'deleteProduct' },
)
```

### What it handles

| OpenAPI feature | Generated output |
|-----------------|-----------------|
| `GET` endpoints | `resource()` with `offline: true` |
| `POST / PUT / PATCH / DELETE` | `action()` with `reliability: 'neverLose'` |
| `components/schemas` objects | TypeScript `interface` |
| `components/schemas` enums / unions | TypeScript `type` alias |
| `$ref` references | Resolved to interface names |
| Path parameters (`{id}`) | Merged into payload type + template literal URL |
| `operationId` | Used as the declaration name (camelCased) |
| No `operationId` | Derived from HTTP method + path segments |
| `summary` | Added as JSDoc comment |
| JSON and YAML specs | Auto-detected by extension (or content sniffing) |

## Workflow

Generated files are meant to be committed and regenerated when your API changes:

```bash
# Add to package.json scripts
"gen:api": "eidos-gen openapi.json --out src/api/eidos.generated.ts"

# Run whenever openapi.json changes
npm run gen:api
```

Edit the **function bodies** freely — change fetch URLs, add headers, transform data. Re-running `eidos-gen` refreshes types and declarations while leaving your customisations intact (only the file is overwritten, not individual declarations).
